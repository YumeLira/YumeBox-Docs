## 系统要求

- Android 7.0 (API 级别 24) 及以上
- 支持的架构：`armeabi-v7a`、`arm64-v8a`、`x86`、`x86_64`

## 下载 YumeBox

<Steps>
  <Step title="访问 Releases 页面">
    访问 [GitHub Releases](https://github.com/YumeYuka/YumeBox/releases/latest) 页面
  </Step>
  <Step title="选择正确的架构版本">
    根据你的设备架构下载对应的 APK 文件：
    - 大多数现代设备使用 `arm64-v8a` 版本
    - 较旧设备可能需要 `armeabi-v7a` 版本
    - 模拟器设备可能需要 `x86` 或 `x86_64` 版本
  </Step>
  <Step title="下载 APK 文件">
    点击下载按钮下载对应的 APK 文件到你的设备
  </Step>
</Steps>

## 安装步骤


<Steps>
  <Step title="下载对应的 APK 文件">
    从 GitHub Releases 下载适合你设备架构的 APK 文件
  </Step>
  <Step title="允许未知来源安装">
    在 Android 设置中允许"未知来源"应用安装：
    - 进入"设置" > "安全与隐私" > "更多安全设置"
    - 启用"安装未知应用"选项
    - 找到你的浏览器应用并允许安装
  </Step>
  <Step title="安装应用">
    打开下载的 APK 文件并按照提示完成安装
  </Step>
  <Step title="启动应用">
    启动 YumeBox 并授予必要的权限（网络、VPN等）
  </Step>
</Steps>

## 初次配置

安装完成后，你需要完成以下基本配置：

<Steps>
  <Step title="导入配置文件">
    选择一种方式导入配置文件：
    - 通过 URL 导入
    - 通过本地文件导入
    - 通过 Clash URL Scheme 导入
  </Step>
  <Step title="设置系统代理">
    允许必要的权限：
    - 授予 VPN 权限以创建系统代理连接
    - 配置网络访问权限
  </Step>
  <Step title="启动服务">
    - 选择代理模式（规则/全局/直连/脚本）
    - 选择可用的代理节点
    - 点击启动按钮启动代理服务
  </Step>
</Steps>

### 外部控制 API

YumeBox 支持通过 Android Intent 进行外部控制，使其他应用能够启动或停止代理服务。

- 启动 Clash.Meta 服务

向活动 `com.github.yumelira.yumebox.MainActivity` 发送带有动作
`com.github.yumelira.yumebox.action.START_CLASH` 的 Intent

- 停止 Clash.Meta 服务

向活动 `com.github.yumelira.yumebox.MainActivity` 发送带有动作
`com.github.yumelira.yumebox.action.STOP_CLASH` 的 Intent

- 导入配置文件

使用 URL Scheme：`clash://install-config?url=<encoded URI>`
或 `clashmeta://install-config?url=<encoded URI>`

## 子功能扩展安装

YumeBox 还提供 SubStore 扩展，提供额外的配置管理功能：

<Steps>
  <Step title="下载扩展">
    从 Releases 页面下载 `YumeBox-Extension-v*.apk` 文件
  </Step>
  <Step title="安装扩展">
    按照相同的安装步骤安装扩展应用
  </Step>
  <Step title="启用扩展">
    在 YumeBox 主应用中启用 SubStore 扩展
  </Step>
</Steps>
